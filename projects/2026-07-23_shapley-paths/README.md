# shapley-paths — 並列推論パスのShapley値報酬帰属

ai-project-loop **Cycle 8** の成果物(2026-07-23)。

## 概要

LLMの並列推論(複数の推論パスを多数決で集約)で、各パスのアンサンブル成功への
**真の貢献度をShapley値で厳密計算**し、貢献者とフリーライダー/誤誘導パスを識別するCLI。
結果報酬を全パスに一律付与する素朴なやり方の代わりに、限界貢献ベースの帰属を与える。
Python 3 標準ライブラリのみ。

- 協力ゲーム: 提携 S の価値 `v(S)` = S の多数決が gold と一致なら 1、それ以外は 0
- Shapley値: `φ_i = Σ |S|!(n-|S|-1)!/n! · (v(S∪{i}) - v(S))`(全順序での限界貢献平均)
- **効率性 `Σφ_i = v(N)`** を満たす → 出力末尾で検算に使える

## 着想元(11_AI Archive)

- [[2026-07-23-fishing-out-free-riders-shapley-based-reward-attribution-for-6e50]] —
  並列推論では結果報酬が全パスに一律割当されるため冗長・誤誘導なパス(フリーライダー)にも
  同じ報酬が付き学習信号が曖昧になる問題に対し、**Shapley値で各パスの貢献を推定**する
  Parallel Shapley の論文。本ツールはその帰属を小規模で厳密計算する実装。

## 使い方

```bash
python shapley_paths.py sample/vote.json
```

- config: `{"gold":"B","answers":["B","B","B","A","C"]}` または `paths:[{id,answer}]`
- `--json` で機械可読出力 / n≤18 は厳密(2^n 提携)

## 動作確認結果(2026-07-23)

`vote.json`(gold=B, 回答 3×B・1×A・1×C):

```
path-1  B   +0.4333  ✓ 貢献者
path-4  A   -0.1500  ✗ 誤誘導(限界貢献が負)
path-5  C   -0.1500  ✗ 誤誘導
Σφ = 1.0000  (= v(N) = 1; 効率性の検算)
```

`close.json`(gold=B, 回答 3×B・2×A, 接戦):

```
p1  B  +0.5333  ✓ 貢献者      p3  A  -0.3000  ✗ 誤誘導
```

3つの正解パスが均等に credit を分け合い、誤ったパスは負のShapley。接戦(close)では
誤誘導パスの害が大きく(-0.30 vs -0.15)、票差が縮むほどノイズパスのコストが増すことも表現。
両ケースで `Σφ = v(N)` が成立し、帰属の正当性を検算できている。

## 制限事項

- 提携価値は「多数決=gold」の 0/1。graded reward や重み付き集約は未対応
- n≤18 は厳密、それ以上はモンテカルロ近似が必要(未実装で警告のみ)
- パスの答えは所与。パス生成やRL学習ループ自体は含まない(帰属の可視化まで)
