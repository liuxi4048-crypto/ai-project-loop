# sdfplan — 符号付き距離関数(SDF)による動作計画

ai-project-loop **Cycle 40** の成果物(2026-07-23)。

## 概要

占有(0/1)地図では衝突判定しかできないが、各セルに「最近傍障害物までの距離」を符号化した
**SDF(クリアランス場)**を使うと余裕を考慮した豊かな計画ができる。本ツールはグリッド地図から
SDF を計算し、A* の移動コストにクリアランスを織り込んで「壁に張り付かず距離を保つ」経路を求め、
素朴な最短経路と比較する。標準ライブラリのみ・決定論的。

## 着想元(11_AI Archive)

- [[2026-07-23-from-distances-to-trajectories-real-time-signed-distance-fun-9c0f]] —
  UAVのマッピングと動作計画を**SDFを軸に共設計**し、最近傍障害物までの距離を符号化して占有のみ
  より豊かな計画情報を得る研究。本ツールはその中核(SDF計算 + 距離を織り込んだ計画)を2Dで実装。

## 使い方

```bash
python sdfplan.py --w 6.0
```

- `--w` クリアランス重み(大きいほど壁を避ける)/ `--json` で機械可読出力

## 動作確認結果(2026-07-23)

中央に障害物ブロックのある 15×11 地図、左S→右G:

```
最短経路(SDF無視):   障害物に張り付く   経路長 17.31   最小クリアランス 1.0
SDF考慮経路(w=6):     外側へ膨らむ       経路長 19.31   最小クリアランス 2.0
```

SDFを移動コストに織り込むと、経路長が僅かに伸びる代わりに最小クリアランスが 1.0→2.0 へ向上。
占有のみの最短経路が壁に張り付くのに対し、SDF考慮は障害物から安全距離を確保する
(ASCIIで経路 `*` を可視化)。

## 制限事項

- SDFは全障害物への総当たりで計算(大規模グリッドは距離変換/Octreeが必要 ― 論文はOctree実装)
- グリッド8近傍A*(連続空間の滑らかな軌道最適化ではない)。動力学・速度制約は未考慮
- 2D静的地図(論文はUAVの3Dリアルタイム逐次マッピング)
