# benchspot — ベンチマーク汚染(特定プロンプト過学習)の検出

ai-project-loop **Cycle 42** の成果物(2026-07-23)。

## 概要

有名な公開ベンチマークは各社がそれ向けに最適化しやすく、汎化指標として機能しなくなる
「ベンチマーク汚染」が起きる。行=題材A・列=題材B のスコアグリッドに**二元加法モデル**
(スコア ≈ 全体平均 + 行効果 + 列効果)を当て、加法的期待を大きく上回る残差の外れセルを
「特定の組合せへ過学習した(汚染)疑い」として検出する。標準ライブラリのみ・決定論的。

## 着想元(11_AI Archive)

- [[2026-07-23-are-ai-labs-pelicanmaxxing-3c14]] —
  「自転車に乗るペリカンを描かせる」有名ベンチに各社が最適化しているのでは、を **動物×乗り物の
  グリッド**で統計検証した記事(裏取り2ソース)。本ツールはその方法論(ペリカン一般・自転車一般の
  上手さを差し引いてなお pelican×bicycle が突出するか)を二元加法モデルの残差検定で実装。

## 使い方

```bash
python make_sample.py                 # 動物8×乗り物6のスコアグリッド生成(pelican×bicycleを汚染)
python benchspot.py sample/pelican_grid.json
```

- 入力: `{rows, cols, scores:[[...]], z_threshold}` / `--json` で機械可読出力
- 終了コード: 汚染疑いあり=1・なし=0

## 動作確認結果(2026-07-23)

```
スコアグリッド 8行×6列  全体平均 5.39  残差z閾値 2.5
汚染疑い: ⚠ pelican × bicycle: 実測 10.14 vs 期待 7.72 (残差 +2.42, z=5.71)
```

ペリカン行効果(+)・自転車列効果(+)を差し引いた加法的期待 7.72 を、実測 10.14 が大きく
上回り(z=5.71)、pelican×bicycle **だけ**を汚染疑いとして検出。他セルは加法モデルに従い非検出。

## 制限事項

- 二元加法モデルの残差外れ値検出(交互作用)。行/列効果が汚染の一部を吸収するため残差は過小評価寄り
- 単一グリッド・単一モデル(複数モデル間の相対比較や反復測定のばらつきは未考慮)
- 「突出=汚染」は疑いであって証明ではない(その組合せが本質的に易しい可能性も残る)
