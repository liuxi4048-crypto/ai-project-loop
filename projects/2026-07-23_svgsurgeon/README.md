# svgsurgeon — SVGの「外科的編集」検証

ai-project-loop **Cycle 23** の成果物(2026-07-23)。

## 概要

SVG編集の質を「**指示された修復が入ったか**」と「**それ以外(保護対象)を壊していないか**」の
両面で検証するCLI。元SVGと編集後SVGを id で対応づけ、修復の適用・保護対象の非破壊・
修復対象でない要素の**巻き添え変更(collateral)**を判定する。標準ライブラリ(xml.etree)のみ。

## 着想元(11_AI Archive)

- [[2026-07-23-vector-bench-can-models-surgically-edit-svg-code-ce44]] —
  SVGを「外科的に編集」できるかを、**検証済み修復**と**保護対象オブジェクト**のペアで測る
  Vector-Bench(40タスク)。本ツールはその採点核「修復が入り、かつ保護対象が無傷か」を実装。
- 系譜: 実行して合否を出す [[2026-07-23-scicodepile-a-128gb-corpus-and-executable-benchmark-for-chal-e2d8|codebench]] と同じ「編集/生成の検証」系だが、こちらは**構造化文書(SVG)の外科性**が対象。

## 使い方

```bash
python svgsurgeon.py sample/task-good.json
```

- タスク: `{original, edited, repairs:[{id,attr,expected}], protected:[id...]}`(パスはtask基準)
- `--json` で機械可読出力 / 終了コード: 外科的成功=0・失敗=1

## 動作確認結果(2026-07-23)

同じ修復(円 c1 の fill を red に)を、保護対象を守った編集と壊した編集で検証:

```
GOOD:  修復 1/1  保護 3/3  巻き添え 0  → ✓ 外科的編集 成功
BAD :  修復 1/1  保護 2/3  巻き添え 1  → ✗ 外科的編集 失敗（#r1 を緑→黄に破壊）
```

どちらも修復自体は入るが、BAD は保護対象 r1 を巻き添えで変更。修復適用と保護非破壊を
分離して評価し、外科的成功/失敗を正しく区別できている。

## 制限事項

- id を持つ要素単位の比較(id 無し要素・要素の追加/削除順序・パスデータ内の微差は別途拡張が必要)
- 属性値は文字列完全一致(`red` と `#ff0000`、空白差、数値表記ゆれは同一視しない)
- 名前空間付きタグはそのまま比較(元と編集後で同じ直列化前提)。視覚的レンダリング差は見ない
