# detiler — タイル分割の継ぎ目アーティファクトをスライディング窓平均で低減

ai-project-loop **Cycle 21** の成果物(2026-07-23)。

## 概要

大規模画像をタイル分割して推論すると、各タイルが独立に事後分布からサンプルされる場合に
タイル境界へ**継ぎ目(不連続)**が生じる。本ツールは 2D 数値場で、独立タイル処理と
**重複スライディングウィンドウの平均**を比較し、境界不連続量の低減を定量化する。
Python 3 標準ライブラリのみ・決定論的(乱数なし・ハッシュで擬似オフセット)。

## 着想元(11_AI Archive)

- [[2026-07-23-switi-quantifying-and-reducing-tiling-artifacts-with-sliding-2799]] —
  タイルが受容野より小さい/各タイルが独立サンプルの場合に境界アーティファクトが出る問題を、
  重複窓予測の平均で「固定の継ぎ目座標に不整合を蓄積させず、シフトした窓全体へ分散」する
  テスト時手法 SWITi。本ツールはその効果を最小構成で再現・定量化。

## 使い方

```bash
python detiler.py --size 24 --tile 8 --stride 4
```

- `--stride` を小さくするほど重複が増える / `--json` で機械可読出力
- 指標: 境界段差(タイル境界を跨ぐ隣接画素の平均段差=継ぎ目量)/ 最大段差

## 動作確認結果(2026-07-23)

```
tile=8 stride=4:  境界段差 0.437→0.259 (-40.8%)   最大段差 -18.8%
tile=8 stride=2:  境界段差 0.641→0.118 (-81.6%)   最大段差 -63.8%
```

スライディング窓平均で継ぎ目(境界段差)が明確に低減し、**重複が多い(ストライドが小さい)
ほど低減幅が大きい**という SWITi の主張どおりの挙動を再現できている。

## 制限事項

- 実画像・実ネットワークではなく、滑らかな真の場+タイル独立オフセットの抽象モデル
- 継ぎ目量は「境界を跨ぐ平均段差」で定義(知覚品質・SSIM 等の指標ではない)
- 重複平均は計算コストが増える(窓数に比例)。品質とコストのトレードオフは扱わない
