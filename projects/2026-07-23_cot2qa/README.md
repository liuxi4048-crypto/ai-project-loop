# cot2qa — 推論チェーン→依存関係付き中間QAレコード変換

ai-project-loop **Cycle 14** の成果物(2026-07-23)。

## 概要

推論チェーン(Chain-of-Thought)を解析して**ステップ間の依存グラフ**を構築し、各中間ステップを
「必要な直前状態を条件に局所回答を予測する」QAレコードへ変換するツール。平坦なCoT教師信号を、
依存関係を明示した段階別の学習データに落とし込む。Python 3 標準ライブラリのみ・決定論的。

## 着想元(11_AI Archive)

- [[2026-07-23-dais-dependency-aware-intermediate-qa-supervision-for-comple-4e0c]] —
  平坦なCoT教師あり学習は「局所的結論が後続の判断をどう支えるか」への信号が乏しいと指摘し、
  推論過程を**依存関係を考慮した段階ごとのQAレコード**に変換する DAIS を提案。本ツールはその
  変換(トレース→依存グラフ→中間QA)をデータ生成として実装。
- 系譜: トレースを解析する [[2026-07-23-copy-less-ground-more-overcoming-repetitive-copying-in-long-d725|copyrate]] / [[2026-07-23-reasoning-error-from-known-fact-step-level-self-consistency-484d|stepcheck]] は「検査」だが、本ツールはトレース→**構造化学習データへの変換**。

## 仕組み

- 依存判定: ステップ間で共有される**数値**と**固有語**をリンク信号にする
- 全ステップに頻出する語(例: "apples")はドメイン語として除外(偽の全結合を防ぐ)
- 明示的な「step N」参照・バックリファレンス("therefore" 等)も依存に加える

## 使い方

```bash
python cot2qa.py sample/trace.json
```

- 入力: `{"question":..., "steps":[...], "answer":...}` / `--json` で機械可読出力

## 動作確認結果(2026-07-23)

算数の文章題(200個中45+90=135売却→65残り)で、演算のデータフロー通りの依存グラフを構築:

```
  step 1 ← (質問のみ)          # 200個
  step 2 ← (質問のみ)          # 45個(朝)
  step 3 ← step 2              # 朝の2倍=90
  step 4 ← step 2, step 3      # 45+90=135
  step 5 ← step 1, step 4      # 200-135=65
中間QAレコード 6 件(依存エッジ 5 本)
```

頻度フィルタ導入前は "apples" が全ステップを誤連結(10エッジ)していたが、除外後は真の依存
5本のみを抽出。各中間QAが「必要な直前状態」だけを条件とする DAIS 構造を再現できている。

## 制限事項

- 依存は表層(数値・語)の重なりヒューリスティック(意味的含意・因果は判定しない)
- 頻度フィルタは df≤n/2 の単純規則。短いチェーンや同義語には調整が要る
- 中間QAの question 文はテンプレート(教師モデルによる問い生成は含まない)
