# feedbackblind — 「観測のみ検索」のフィードバック盲目性の実証

ai-project-loop **Cycle 19** の成果物(2026-07-23)。

## 概要

観測の類似性だけで過去履歴を検索する意思決定は、**観測は同一だが行動・報酬の結果が異なる
履歴を区別できない**(フィードバック盲目的検索)。本ツールは、成功デモの軌跡群を記憶に持ち、
「現在のレジームを示す最近の(観測,行動,報酬)」を手がかりに、次の2方式で行動を選んで比較する:

- **obs-only(盲目)**: 観測一致の成功行動を全軌跡から多数決 → レジーム衝突で誤答
- **utility-aware**: 最近の報酬文脈に整合する軌跡へ絞ってから検索 → 正答

Python 3 標準ライブラリのみ・決定論的(乱数なし)。

## 着想元(11_AI Archive)

- [[2026-07-23-breaking-feedback-blindness-utility-augmented-transformer-fo-9a22]] —
  Transformer型決定モデルのアテンション検索が観測由来の類似性に駆動され、観測が同一でも
  行動・報酬が異なる履歴を区別できない構造的ボトルネックを「フィードバック盲目的検索」と
  定式化した論文。本ツールはその失敗と、効用(報酬)条件付けによる解消を最小構成で実証。

## 使い方

```bash
python feedbackblind.py
```

- `--json` で機械可読出力。設定(レジーム・観測・正解行動)はコード先頭の `CORRECT` を編集

## 動作確認結果(2026-07-23)

2レジーム(同じ観測で正解行動が入れ替わる)、成功デモ軌跡6本、クエリ4件:

```
 regime  query      context   正解  obs_only  utility
      A     o2    o1→a0(r1)    1       0 ✗        1
      B     o1    o2→a0(r1)    1       0 ✗        1
-- obs_only 正答率 50%(観測が衝突すると盲目)  /  utility_aware 正答率 100%
```

obs_only は各観測で両行動が同数得票し多数決が割れて 50%(=二択の当て推量)。utility_aware は
報酬文脈が示す現在レジームの軌跡に絞ることで、同一観測でも正しい行動を 100% 選べている。

## 制限事項

- 概念実証の最小構成(2レジーム・2観測・決定論デモ)。実環境の非定常性や探索は含まない
- utility-aware の「レジーム推定」は文脈ステップの厳密一致による軌跡フィルタ(近似・ノイズ耐性は未実装)
- 実モデル(Transformer)ではなく、検索方式の対比を切り出した抽象デモ
