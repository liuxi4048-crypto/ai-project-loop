# merkle — Merkleツリーによる完全性検証・改竄検出

ai-project-loop **Cycle 48** の成果物(2026-07-23)。

## 概要

学習データやモデル成果物の完全性・来歴を担保するには、集合全体を1つのハッシュ(ルート)に
まとめ、個々の要素をルートだけから検証できると都合がよい。**Merkleツリー**は各チャンクの
ハッシュを二分木状に結合し、ルート1個で全体を代表する。任意の1要素は**包含証明
(inclusion proof)** = 兄弟ハッシュの列(O(log n)個)を辿ってルート一致を検証でき、どこか
1バイトでも改竄するとルートが変わる。標準ライブラリのみ(sha256)・決定論的。
葉と内部ノードをドメイン分離(0x00/0x01)して第二原像攻撃を防ぐ。

## 着想元(11_AI Archive)

- 完全性・来歴の横断テーマ(横断フォールバック): グリーンリスト透かし
  [[2026-07-22-meta-made-its-own-ai-detection-system-it-should-have-just-us-cd49|SynthID/AI検出]] や、

## 使い方

```bash
python merkle.py            # 組込みデモ(構築→包含証明→改竄検出)
python merkle.py <file>     # ファイルを行単位でチャンク化して検証
```

## 動作確認結果(2026-07-23)

```
チャンク 6件  木の高さ 4段
包含証明: 6/6 件がルートに整合(全要素を検証)
  例: 要素0の証明長 = 3 ハッシュ(全6件でなく log₂ 個で検証)
改竄検出: 要素3を1バイト改竄 → ルート変化 ✓  改竄要素は旧ルートに不整合 ✓
```

ルート1個で集合全体の完全性を代表。全要素の包含証明が整合し、1要素の1バイト改竄で必ず
ルートが変化・旧ルートに不整合となる。各要素は log₂ 個の兄弟ハッシュだけで検証できる。

## 制限事項

- 静的な集合の検証(追記可能な Merkle Mountain Range / 履歴の一貫性証明は未実装)
- 奇数段は末尾ハッシュを複製(RFC6962 の別方式とはルート値が異なる)
- ハッシュはSHA-256固定。証明の署名・タイムスタンプ等の来歴付与は範囲外
