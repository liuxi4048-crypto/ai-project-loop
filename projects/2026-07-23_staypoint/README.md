# staypoint — GPS軌跡からの滞在点検出

ai-project-loop **Cycle 35** の成果物(2026-07-23)。

## 概要

生のGPS軌跡から**滞在点(staypoint)**を検出し、座標列を「自宅・職場・店」といった意味ある
場所へ変換するCLI。距離閾値 D 内に時間閾値 T 以上とどまった連続点群を1つの滞在点(重心)に
まとめる古典手法(Li et al.)を実装。標準ライブラリのみ・決定論的(haversine距離)。

## 着想元(11_AI Archive)

- [[2026-07-23-staypoint-detection-from-noisy-trajectory-data-experiment-pa-4114]] —
  生の軌跡から滞在点を検出し**意味ある場所へ変換**する処理のベンチマーク論文(標準ベンチ不在の
  ギャップを指摘)。本ツールはその中核アルゴリズム(距離・時間閾値による滞在点検出)を実装。

## 使い方

```bash
python make_sample.py                 # 自宅→移動→カフェ のノイズ入り軌跡を生成
python staypoint.py sample/trip.json
```

- 入力: `{distance_m, time_s, points:[{lat,lon,t}]}` / `--json` で機械可読出力

## 動作確認結果(2026-07-23)

自宅滞在→移動→カフェ滞在の16点軌跡(D=50m, T=300s):

```
  滞在点1: (34.9999, 139.0000)  到着 t=0s   出発 t=600s   滞在 10分 (7点)   ← 自宅
  滞在点2: (35.0099, 139.0100)  到着 t=900s  出発 t=1500s  滞在 10分 (7点)   ← カフェ
-- 滞在点 2件 / 滞在点に属する点 14 / 移動点 2
```

ノイズを含む生座標列から、真の自宅・カフェ位置に一致する2つの滞在点を抽出し、移動中の点
(2点)を分離。生軌跡を意味ある場所の系列へ変換できている。

## 制限事項

- アンカー点からの距離判定(Li et al.)。クラスタの直径ではなく先頭点基準のため、緩やかな
  ドリフトは1滞在点に含まれやすい(密度ベースのDBSCAN系は別実装が必要)
- 距離・時間閾値は所与。実データでは移動速度・サンプリング間隔に応じた調整が要る
- 滞在点の意味付け(自宅/職場のラベリング)は別処理(頻度・時間帯の分析が必要)
