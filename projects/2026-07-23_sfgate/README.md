# sfgate — 統計優先ゲーティング(信頼区間で確定/エスカレート)

ai-project-loop **Cycle 36** の成果物(2026-07-23)。

## 概要

品質をノイズ入りの安価な計測で推定するとき、平均の**95%信頼区間(CI)**を用いて判定する
「統計優先ゲート」。CIが閾値を明確に上回れば**受理**、明確に下回れば**却下**、閾値をまたぐ/
標本不足/CIが広すぎる場合は「**審議へエスカレート**」する。点推定でなく区間で判断し、
確信できる時だけ確定する。標準ライブラリのみ・決定論的。

## 着想元(11_AI Archive)

- [[2026-07-23-sfga-a-statistics-first-gating-architecture-with-adjudicativ-482c]] —
  SFTデータ調達を、各品質軸を**信頼区間付きで推定し、区間が十分狭く標本が足り閾値に一致した
  時のみ確定、さもなくば審議(adjudicative)へエスカレート**する統計優先ゲーティング SFGA。
  本ツールはその判定核(CIベースの確定/エスカレーション)を実装。

## 使い方

```bash
python sfgate.py sample/corpora.json
```

- 入力: `{threshold, min_samples, max_ci_halfwidth, candidates:[{id,measurements:[...]}]}`
- `--json` で機械可読出力

## 動作確認結果(2026-07-23)

閾値0.70で5候補(各品質計測):

```
   src_A  n6 0.825 ±0.015  [✓受理]  CI下限 0.810 > 0.7
   src_B  n6 0.505 ±0.015  [✗却下]  CI上限 0.520 < 0.7
   src_C  n6 0.713 ±0.146  [→審議]  CIが広すぎる（ノイズ大）
   src_D  n1 0.900 ±∞      [→審議]  標本不足
   src_E  n6 0.722 ±0.006  [✓受理]  CI下限 0.716 > 0.7（僅差だが高精度）
```

平均が閾値を上回る src_C でも CI が広ければ確定せず審議へ回し、平均が僅差の src_E でも CI が
極めて狭ければ確信して受理。**点推定でなく統計的確信度で判断**するSFGAの論理を再現できている。

## 制限事項

- 95%CIは正規近似(小標本は t 分布/ブートストラップが望ましい)
- 単一品質軸の判定(SFGAの多様性・有用性・冗長性の多軸一致判定は未実装)
- エスカレーション先の「審議(adjudicative debate)」自体は含まない(ゲート判定まで)
