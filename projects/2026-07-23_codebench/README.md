# codebench — 実行型コード生成ベンチマークランナー

ai-project-loop **Cycle 18** の成果物(2026-07-23)。

## 概要

候補解プログラムを **I/Oテストケースに対して実際に実行**し、問題別のテスト通過数と全体の
**pass@1**(全テスト通過した問題の割合)を集計する実行型ベンチマークランナー。
標準入力にテスト入力を与え、標準出力を期待値と照合する。Python 3 標準ライブラリのみ。

> ⚠ 与えられたコードを subprocess で実行する。**信頼できる自分のサンプルにのみ**使うこと
> (任意コード実行の隔離はしていない)。

## 着想元(11_AI Archive)

- [[2026-07-23-scicodepile-a-128gb-corpus-and-executable-benchmark-for-chal-e2d8]] —
  科学コード生成を「動くか」で測る**実行可能ベンチマーク**(37,737リポジトリ・128GB)を
  提示した論文。本ツールはその中核「解を実行して合否判定する」を最小構成で実装。

## 使い方

```bash
python codebench.py bench.json
```

- 入力: `{"problems":[{id, solution_file, tests:[{input, expected}]}]}`(パスは bench.json 基準)
- `--timeout` 秒 / `--json` 出力 / 終了コード: 全問full pass=0・不合格あり=1

## 動作確認結果(2026-07-23)

正しい解(add)とバグ入りの解(max=誤って min を出力)の2問:

```
  ✓ add          3/3 tests
  ✗ max          0/2 tests   test1: got '3' want '7'
-- pass@1 = 50.0% (1/2 問題が全テスト通過)
```

正しい解は全テスト通過、バグ入りは不合格として差分(got/want)を明示。異常終了・タイム
アウトも失敗として扱う。実際にコードを走らせて合否を判定できている。

## 制限事項

- 任意コード実行のサンドボックス化はしない(信頼できる入力前提)。timeout のみ保護
- 判定は標準出力の完全一致(strip後)。浮動小数の許容誤差・複数正解には未対応
- 標準入力→標準出力のバッチ形式のみ(対話・ファイルI/O・引数は別途拡張が必要)
