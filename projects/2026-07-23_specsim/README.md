# specsim — 投機的デコーディングの高速化見積り(固定γ vs 適応γ)

ai-project-loop **Cycle 22** の成果物(2026-07-23)。

## 概要

投機的デコーディング(軽量ドラフタが γ トークンの下書き→ターゲットが並列検証・受理)の
高速化を閉形式で見積り、**単一の固定ドラフト長 γ** と **文脈ごとに最適 γ を選ぶ適応方式** を
比較するシミュレータ。標準ライブラリのみ・決定論的。

- 1ブロックの期待受理トークン: `E[k] = (1 − α^(γ+1)) / (1 − α)`(α=受理確率, 追加1トークン込み)
- ブロックコスト: `1(並列検証) + γ·c(ドラフタ相対コスト)`
- 高速化 = 受理トークン / コスト(自己回帰=1x 基準)

## 着想元(11_AI Archive)

- [[2026-07-23-adaflash-adaptive-speculative-decoding-via-on-policy-distill-1e48]] —
  拡散ドラフターの双方向アテンションが「並列生成・広文脈」という利点と欠点を併せ持つ諸刃の剣
  であることを指摘し、**適応的に振る舞いを切り替える** AdaFlash を提案。本ツールはその核心
  「文脈適応が固定を上回る」を、投機デコードの標準的な高速化式で定量再現。

## 使い方

```bash
python specsim.py --c-draft 0.15 --max-gamma 8
```

- `--c-draft` ドラフタ1トークンの相対コスト / `--json` で機械可読出力

## 動作確認結果(2026-07-23)

受理率 α が変動する12文脈:

```
c_draft=0.15:  適応 α0.9→γ8 / α0.3→γ1 …   最良固定γ=5: 1.827x   適応: 2.202x (+20.5%)
c_draft=0.30:  ドラフタが高コスト→最適γ縮小  最良固定γ=2: 1.349x   適応: 1.592x (+18.0%)
```

適応は**高α区間で γ を伸ばし、低α区間で削る**ことで、いかなる単一固定 γ も超える実効
スループットを達成。ドラフタコストが上がると最適 γ が小さくなる挙動も再現できている。

## 制限事項

- 解析モデル(受理確率 α とドラフタ相対コストのパラメータ化)。実トークナイザ・実測ロジットは扱わない
- α は文脈ごとに既知と仮定(実運用は α をオンラインで推定する必要がある)
- 受理はトークン独立と仮定(実際は系列内で相関し、拡散ドラフタ特有の効果は別途モデル化が必要)
